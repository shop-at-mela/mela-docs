# Sitemap & Robots.txt PRD

**Feature:** Sitemap completeness + robots.txt production readiness
**Status:** Ready for Development
**Priority:** High — Pre-Production (SEO)
**Owner:** Developer

---

## Current State

The sitemap is dynamically generated by `server/resources/sitemap.js`. The core infrastructure is solid — 4 sitemaps are generated (index, default, listings, pages) with 1-day caching. However, several Mela-specific routes are missing, and the robots.txt has a placeholder domain.

---

## Gap Analysis

### Gap 1: `robots.txt` Has Placeholder Domain

**File:** `server/resources/robots.txt` line 23

```
# Current (WRONG — will break Google Search Console verification):
Sitemap: https://my.marketplace.com/sitemap-index.xml

# Fix:
Sitemap: https://[production-domain]/sitemap-index.xml
```

**This is a one-line fix but a hard launch blocker.** Google Search Console will not verify a sitemap at a placeholder domain.

**Action:** Confirm production domain, update line 23 before deploy.

---

### Gap 2: Mela Custom Routes Missing from `sitemap-default.xml`

`defaultPublicPaths` in `server/resources/sitemap.js` (approx. lines 46–53) currently only includes:

```javascript
'/', '/terms-of-service', '/privacy-policy', '/signup', '/login', '/s'
```

These high-value Mela routes are **not included**:

| Route | Path | SEO Value |
|-------|------|-----------| 
| BrandsPage | `/brands` | High — directory page, linkable |
| CategoriesPage | `/categories` | High — SEO category hub |

**Fix:** Add `/brands` and `/categories` to `defaultPublicPaths`. One-line change.

---

### Gap 3: Brand Detail Pages Not in Sitemap

Brand storefronts at `/brands/:brandSlug` and `/u/:id` are high-SEO targets — each brand is a keyword-rich page ("Organic cotton baby brand India"). Neither URL pattern is currently in any sitemap.

**Fix:** Add a new `sitemap-brands.xml` that queries published seller profiles via the Integration API and generates URLs for:
- `/brands/:brandSlug` (from `user.attributes.profile.publicData.brandSlug`)
- `/u/:id` (canonical profile URL)

This follows the same pattern as `sitemap-recent-listings.xml`.

---

### Gap 4: Category Detail Pages Not in Sitemap

Category pages at `/categories/:level1/:level2` are static (derived from the config), not from the SDK. They should be in `sitemap-default.xml` as hardcoded paths.

**Fix:** Enumerate all category paths from `configListing.js` category hierarchy and add to `defaultPublicPaths`. Since categories are static config (not user-generated), this is a one-time enumeration, not a dynamic query.

---

## Recommended Sitemap Structure (Production)

```
sitemap-index.xml
├── sitemap-default.xml         (static routes: /, /brands, /categories, /categories/*, ToS, PP, etc.)
├── sitemap-brands.xml          (NEW: /brands/:brandSlug + /u/:id per published seller)
├── sitemap-recent-listings.xml (existing: /l/:uuid — up to 10,000)
└── sitemap-recent-pages.xml    (existing: /p/:pageId CMS pages)
```

Note: `sitemap-categories.xml` is not needed separately — category paths are static config and fit in `sitemap-default.xml`.

## Implementation Plan

  '/categories': { changefreq: 'weekly', priority: 0.8 },

### Step 3: Enumerate category paths into `sitemap-default.xml` (1–2 hrs)

- Generate static paths: `/categories/:level1`, `/categories/:level1/:level2`
- Add all to `defaultPublicPaths` as hardcoded strings (categories don't change at runtime)

### Step 4: Add `sitemap-brands.xml` (half day)
- New function in `server/resources/sitemap.js` following `buildSitemapRecentListings` pattern

- Generate URLs: `/brands/:brandSlug` and `/u/:id` per seller
- Register in `sitemap-index.xml` builder
- Apply same 1-day cache TTL as other sitemaps

### Step 5: Verify after deploy
- Hit `/sitemap-index.xml` — confirm all 4 sitemaps are listed with correct domain
- Hit each sitemap — confirm URLs resolve to 200
- Submit `sitemap-index.xml` URL to Google Search Console
- Confirm robots.txt is accessible at `/robots.txt` and shows production domain

## Out of Scope

- Profile pages for buyer accounts (`/u/:id` for non-sellers) — excluded intentionally, no SEO value
- Dynamic search result URLs — excluded per SEO best practices (duplicate content risk)
- `/styleguide/*` routes — already excluded from robots.txt Disallow rules
